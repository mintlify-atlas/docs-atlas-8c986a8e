## Overview

The `CaptureCoordinator` class is the central hub that receives normalized network events, maintains in-memory transactions, orchestrates inference engines (endpoint/schema/openapi), and emits updates to connected consumers.

### Responsibilities

- Normalize incoming events from multiple sources
- Maintain request/response transactions
- Run inference pipelines and update OpenAPI spec
- Optionally produce embeddings for storage

## Constructor

```typescript
new CaptureCoordinator(options: CaptureCoordinatorOptions)
```

<ParamField path="options" type="CaptureCoordinatorOptions" required>
  Configuration options for the coordinator
  
  <ParamField path="options.aiEnabled" type="boolean" required>
    Enable AI-powered schema refinement
  </ParamField>
  
  <ParamField path="options.fullCapture" type="boolean">
    When true, prefer Playwright/CDP full resource capture and ignore injected script events
  </ParamField>
</ParamField>

## Methods

### handleNetworkEvent

Handle incoming network event from browser.

```typescript
handleNetworkEvent(event: any): void
```

<ParamField path="event" type="object" required>
  Raw event from network capture script
  
  <ParamField path="event.type" type="string" required>
    Event type: `"request"`, `"response"`, `"websocket:send"`, or `"websocket:receive"`
  </ParamField>
  
  <ParamField path="event.data" type="any" required>
    Event data (varies by type)
  </ParamField>
  
  <ParamField path="event.timestamp" type="number" required>
    Event timestamp in milliseconds
  </ParamField>
  
  <ParamField path="event.source" type="string">
    Event source: `"injected"` or `"playwright"`
  </ParamField>
</ParamField>

**Example:**

```typescript
const coordinator = new CaptureCoordinator({
  aiEnabled: true,
  fullCapture: false
});

coordinator.handleNetworkEvent({
  type: "request",
  data: {
    id: "req-123",
    url: "https://api.example.com/users/42",
    method: "GET",
    headers: { "Authorization": "Bearer token" },
    body: null
  },
  timestamp: Date.now(),
  source: "playwright"
});
```

## Events

The `CaptureCoordinator` extends `EventEmitter` and emits the following events:

### request

Emitted when a new request is captured.

```typescript
coordinator.on("request", (request: NetworkRequest) => {
  console.log(`Request: ${request.method} ${request.url}`);
});
```

### response

Emitted when a response is captured.

```typescript
coordinator.on("response", (response: NetworkResponse) => {
  console.log(`Response: ${response.status}`);
});
```

### openapi:update

Emitted when the OpenAPI specification is updated.

```typescript
coordinator.on("openapi:update", (spec: OpenAPISpec) => {
  console.log(`Spec updated with ${Object.keys(spec.paths).length} paths`);
});
```

### websocket

Emitted for WebSocket events.

```typescript
coordinator.on("websocket", (event: { type: string; data: any }) => {
  console.log(`WebSocket: ${event.type}`);
});
```

## Internal behavior

### Request filtering

The coordinator automatically filters out:

- Next.js router prefetch requests (header: `next-router-prefetch`)
- React Server Components requests (header: `rsc`)
- Internal framework assets (`/_next`, `/static`, sourcemaps, etc.)

### Inference pipeline

When a transaction completes, the coordinator:

1. Infers endpoint pattern using `EndpointInferenceEngine`
2. Infers request/response schemas using `SchemaInferenceEngine`
3. Updates OpenAPI spec using `OpenAPIGenerator`
4. Emits `openapi:update` event
5. Optionally creates embeddings for AI features (when `aiEnabled` is true)

### Full capture mode

When `fullCapture` is enabled, the coordinator:

- Prefers Playwright/CDP-sourced events
- Ignores injected-page events to avoid duplicates
- Provides more complete network capture including binary resources
