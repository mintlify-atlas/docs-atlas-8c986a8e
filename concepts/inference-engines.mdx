## Endpoint inference engine

**Location:** `src/dev-tools/core/endpoint-inference-engine.ts`

This engine analyzes URL patterns across multiple requests to identify stable endpoint structures and detect variable path segments.

### Pattern detection workflow

<Steps>
  <Step title="Parse URL">
    Extract path segments and query parameters from the request URL
  </Step>
  <Step title="Group similar paths">
    Cluster URLs with the same number of segments (structural similarity)
  </Step>
  <Step title="Detect variables">
    Identify segments that vary across requests (IDs, UUIDs, slugs)
  </Step>
  <Step title="Generate pattern">
    Replace variable segments with parameterized placeholders like `{id}`
  </Step>
</Steps>

### URL parsing

The engine parses URLs to extract components:

```typescript
// src/dev-tools/core/endpoint-inference-engine.ts:93
private parseURL(url: string): ParsedURL {
  try {
    const urlObj = new URL(url);
    const pathSegments = urlObj.pathname.split("/").filter(Boolean);
    const queryParams: Record<string, string> = {};
    
    urlObj.searchParams.forEach((value, key) => {
      queryParams[key] = value;
    });
    
    return {
      baseUrl: urlObj.origin,
      path: urlObj.pathname,
      pathSegments,
      queryParams
    };
  } catch {
    // Handle relative URLs gracefully
  }
}
```

### Parameter detection heuristics

The engine uses pattern matching to identify path parameters:

  <Tab title="Numeric IDs">

    ```typescript
    // Detects: /api/user/123, /api/user/456

    ```
    **Result:** `/api/user/{id}`

  <Tab title="UUIDs">

    ```typescript
      /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(v)

      return true;
    }
    ```
    **Result:** `/api/order/{id}`
  </Tab>
  <Tab title="Alphanumeric slugs">

    ```typescript
    if (values.every((v) => /^[a-z0-9_-]+$/i.test(v) && v.length > 5)) {
      return true;
    }
    ```
    **Result:** `/posts/{param0}`
  </Tab>
</Tabs>

### Pattern comparison algorithm

For each position in the URL path, the engine compares values across all samples:

```typescript
// src/dev-tools/core/endpoint-inference-engine.ts:202
for (let i = 0; i < segmentCount; i++) {
  const values = segments.map((s) => s[i]);
  const uniqueValues = new Set(values);
  
  if (uniqueValues.size === 1) {
    // All same - keep as literal segment
    pattern.push(values[0]);
  } else if (this.looksLikeParameter(values)) {
    // Variable - convert to parameter
    const paramName = this.inferParameterName(values, i);
    pattern.push(`{${paramName}}`);
    params.push(paramName);
  } else {
    // Mixed - keep literal for now
    pattern.push(values[0]);
  }
}
```

### Stability scoring

Endpoints are assigned stability scores based on the ratio of literal to variable segments:

```typescript
const confidence = 1 - variableSegments / segmentCount;
```

- **1.0** - All segments are literal (e.g., `/api/users`)
- **0.75** - One variable in 4 segments (e.g., `/api/users/{id}/profile`)
- **0.5** - Half variable (e.g., `/api/{resource}/{id}`)

<Note>
  Higher stability scores indicate more reliable endpoint patterns. Patterns with very low stability may need manual review.
</Note>

## Schema inference engine

**Location:** `src/dev-tools/core/schema-inference-engine.ts`

Generates JSON Schema-like structures by analyzing request and response bodies using deterministic heuristics and optional AI enhancement.

### Inference strategy

The engine uses recursive type detection:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:103
private inferSchema(value: any, context: string): JSONSchema | null {
  if (value === null || value === undefined) {
    return { type: "null", nullable: true };
  }
  
  const valueType = typeof value;
  
  // Primitive types
  if (valueType === "string") return { type: "string" };
  if (valueType === "number") {
    return { type: Number.isInteger(value) ? "integer" : "number" };
  }
  if (valueType === "boolean") return { type: "boolean" };
  
  // Complex types (arrays, objects)
  // ...
}
```

### Array schema merging

For arrays, the engine samples the first 5 items and merges their schemas:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:140
if (Array.isArray(value)) {
  if (value.length === 0) {
    return { type: "array", items: { type: "object" } };
  }
  
  const itemSchemas = value
    .slice(0, 5)
    .map((item) => this.inferSchema(item, context));
  const mergedItemSchema = this.mergeSchemas(itemSchemas);
  
  return {
    type: "array",
    items: mergedItemSchema || { type: "object" }
  };
}
```

This handles heterogeneous arrays by finding the common schema structure.

### Object property inference

For objects, each property is recursively analyzed:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:156
Object.entries(value).forEach(([key, val]) => {
  const fieldSchema = this.inferSchema(val, `${context}.${key}`);
  if (fieldSchema) {
    properties[key] = fieldSchema;
    
    // Track field statistics
    this.updateFieldStats(`${context}.${key}`, val);
    
    // Mark as required if not null/undefined
    if (val !== null && val !== undefined) {
      required.push(key);
    }
  }
});

return {
  type: "object",
  properties,
  required: required.length > 0 ? required : undefined
};
```

### Field statistics tracking

The engine maintains statistics for pattern detection:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:240
interface FieldStats {
  types: Map<string, number>;     // Type frequency distribution
  nullCount: number;               // How often field is null
  totalCount: number;              // Total observations
  values: Set<any>;               // Unique values (for enum detection)
  nested?: Map<string, FieldStats>; // Nested object stats
}
```

These statistics enable:
- **Nullable detection** - Fields that are sometimes null
- **Enum detection** - Fields with limited unique values
- **Type consistency** - Detecting polymorphic fields

### Schema merging algorithm

When multiple samples are observed, schemas are merged:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:184
private mergeSchemas(schemas: (JSONSchema | null)[]): JSONSchema | null {
  const validSchemas = schemas.filter((s): s is JSONSchema => s !== null);
  if (validSchemas.length === 0) return null;
  
  // Find most common type
  const types = validSchemas.map((s) => s.type);
  const mostCommonType = this.mostCommon(types);
  
  const merged: JSONSchema = { type: mostCommonType };
  
  // If all objects, merge properties
  if (mostCommonType === "object") {
    const allProperties = new Map<string, JSONSchema[]>();
    
    validSchemas.forEach((schema) => {
      if (schema.properties) {
        Object.entries(schema.properties).forEach(([key, value]) => {
          if (!allProperties.has(key)) {
            allProperties.set(key, []);
          }
          allProperties.get(key)!.push(value);
        });
      }
    });
    
    // Recursively merge each property
    merged.properties = {};
    allProperties.forEach((schemas, key) => {
      const mergedProp = this.mergeSchemas(schemas);
      if (mergedProp) {
        merged.properties![key] = mergedProp;
      }
    });
  }
  
  return merged;
}
```

### AI-powered refinement

When AI is enabled, schemas are enhanced using GPT-4o-mini:

```typescript
// src/dev-tools/core/schema-inference-engine.ts:290
private async refineWithAI(schemas: InferredSchemas): Promise<InferredSchemas> {
  const prompt = `You are a schema inference expert. Improve these schemas by:
1. Detecting enums (if values are limited and repeated)
2. Adding concise descriptions for fields
3. Suggesting clearer field names where obvious
4. Detecting patterns (email, URL, ISO date, UUID, etc)

Request Schema:
${JSON.stringify(schemas.request, null, 2)}

Response Schema:
${JSON.stringify(schemas.response, null, 2)}

Return ONLY valid JSON with improved schemas.`;
  
  const { text } = await generateTextFromGateway({
    model: "openai/gpt-4o-mini",
    prompt
  });
  
  const refined = JSON.parse(text);
  return {
    request: refined.request || schemas.request,
    response: refined.response || schemas.response
  };
}
```

<Warning>
  AI refinement can fail due to network issues or rate limits. The engine always falls back to deterministic schemas if AI refinement fails.
</Warning>

This handles cases where APIs return JSON without proper headers.

```typescript
addEndpoint(input: AddEndpointInput): void {
  const { endpoint, method, requestSchema, responseSchema, headers, statusCode } = input;
  
  // Initialize path if new
  if (!this.spec.paths[endpoint.pattern]) {
    this.spec.paths[endpoint.pattern] = {};

      parameters: [],
      responses: {}

  }
```

### Operation ID generation

Generates developer-friendly operation IDs:

```typescript
// src/dev-tools/core/openapi-generator.ts:288
private generateOperationId(path: string, method: string): string {
  const segments = path
    .split("/")
    .filter(Boolean)
    .map((s) => s.replace(/[{}]/g, ""))  // Remove parameter braces
    .map((s) => s.charAt(0).toUpperCase() + s.slice(1)); // Capitalize
  
  return `${method.toLowerCase()}${segments.join("")}`;
}
```

**Examples:**
- `GET /api/users/{id}` → `getUsersId`
- `POST /api/orders` → `postOrders`
- `DELETE /api/products/{productId}` → `deleteProductsProductId`

## Inference pipeline integration

The three engines work together in sequence:

```typescript
// src/dev-tools/core/capture-coordinator.ts:188
private async processTransaction(transaction: NetworkTransaction): Promise<void> {
  // 1. Infer endpoint pattern
  const endpoint = this.endpointEngine.inferEndpoint({
    url: request.url,
    method: request.method
  });
  
  // 2. Infer schemas
  const schemas = await this.schemaEngine.inferSchemas({
    requestBody: request.body,
    responseBody: response.body,
    contentType: response.headers["content-type"]
  });
  
  // 3. Update OpenAPI spec
  this.openapiGenerator.addEndpoint({
    endpoint,
    method: request.method,
    requestSchema: schemas.request,
    responseSchema: schemas.response,
    headers: request.headers,
    statusCode: response.status
  });
  
  // 4. Broadcast updated spec
  this.emit("openapi:update", this.openapiGenerator.getSpec());
}
```
